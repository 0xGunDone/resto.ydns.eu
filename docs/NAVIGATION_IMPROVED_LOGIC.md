# Улучшенная логика навигации

## Проблемы текущей структуры

1. **"Управление" содержит только "Типы смен"** - не логично
2. **Типы смен должны быть частью управления рестораном** - они привязаны к ресторану
3. **Неочевидная структура** - непонятно, куда попадают разделы

## Предлагаемая структура

### Вариант 1: Упрощенный (рекомендуется)

```
ОСНОВНОЕ:
├─ Главная
└─ Рестораны (OWNER/ADMIN)

РАБОТА:
├─ График
├─ Задачи
└─ Табели

УПРАВЛЕНИЕ:
└─ Управление рестораном → /restaurants/{id}/manage
   (на странице управления есть вкладки:
    - Сотрудники
    - Должности
    - Отделы
    - Типы смен)
```

### Вариант 2: С выпадающим меню

```
УПРАВЛЕНИЕ:
├─ Управление рестораном → /restaurants/{id}/manage
│  └─ Сотрудники
│  └─ Должности
│  └─ Отделы
└─ Типы смен → /shift-types?restaurantId={id}
```

## Логика

1. **Типы смен привязаны к ресторану** - значит должны быть в управлении рестораном
2. **"Управление" = один пункт** - ведет на страницу управления рестораном
3. **На странице управления** - все разделы: сотрудники, должности, отделы, типы смен

## Реализация

### Вариант A: Добавить вкладку "Типы смен" в RestaurantManagePage

Страница `/restaurants/{id}/manage` будет иметь вкладки:
- Сотрудники
- Должности
- Отделы
- **Типы смен** (новое)

### Вариант B: Оставить отдельную страницу, но логично организовать

- "Управление" → `/restaurants/{id}/manage` (основное управление)
- "Типы смен" в подменю "Управление" → `/shift-types?restaurantId={id}`

## Рекомендация

**Вариант A** - добавить вкладку "Типы смен" в `RestaurantManagePage`, так как:
- ✅ Типы смен привязаны к ресторану
- ✅ Логично все управление в одном месте
- ✅ Проще навигация
- ✅ Меньше пунктов меню
