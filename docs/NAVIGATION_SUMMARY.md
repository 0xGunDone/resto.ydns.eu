# Краткое резюме: Структура навигации

## Текущая ситуация

- Плоская структура меню
- Нет группировки разделов
- Неочевидно, для какого ресторана разделы
- Сложно понять зависимости от прав

## Предлагаемое решение

### Структура меню (простая и логичная)

```
ОСНОВНОЕ:
  • Главная (всегда)
  • Рестораны (OWNER/ADMIN)

РАБОТА (требует ресторан):
  • График
  • Задачи
  • Табели

УПРАВЛЕНИЕ (требует права):
  • Управление ▼
    - Типы смен
```

### Логика отображения

1. **Автоматический выбор ресторана**:
   - 1 ресторан → автоматически
   - Несколько → селектор
   - OWNER/ADMIN → селектор всех

2. **Фильтрация по правам**:
   - OWNER/ADMIN → все разделы
   - Другие → только с правами

3. **Контекст ресторана**:
   - Рабочие разделы показываются только при выбранном ресторане
